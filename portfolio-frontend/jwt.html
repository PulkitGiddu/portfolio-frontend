JWT Authentication in Spring Boot

A complete guide covering what JWT is, how it works, all the components involved, and how to implement it from scratch.

1. What is JWT?

JWT (JSON Web Token) is a way to securely transmit information between a client and server as a compact, self-contained string. Once the server issues a JWT, the client sends it with every request — the server just validates it without needing to store any session data.

  • Stateless — the server stores nothing; all info is inside the token
  • Self-contained — the token carries user info (username, roles)
  • Signed — tamper-proof; server verifies the signature on every request
Simple analogy: Think of JWT like a hotel key card. The hotel (server) issues it at check-in (login). Every time you use it to open a door (API call), the door just checks if the card is valid — no need to call the front desk every time.

2. Token Structure

A JWT is three Base64-encoded parts separated by dots:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9         <-- Header
.
eyJzdWIiOiJhbGljZSIsInJvbGVzIjpbIlVTRVIiXSwiZXhwIjoxNzAwMDAzNjAwfQ  <-- Payload
.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c  <-- Signature

Part 1 — Header

Describes the token type and signing algorithm.

{
  "alg": "HS256",   // algorithm used to sign
  "typ": "JWT"
}

Part 2 — Payload (Claims)

Contains the actual data (who the user is, when it expires, etc.).

{
  "sub": "alice",             // subject = username
  "roles": ["ROLE_USER"],     // user's roles
  "iat": 1700000000,          // issued at (unix timestamp)
  "exp": 1700003600           // expires at (1 hour later)
}
Important: The payload is only Base64-encoded, NOT encrypted. Anyone can decode it. Never put passwords, credit cards, or sensitive data in the payload.

Part 3 — Signature

This is how the server verifies the token was not tampered with.

HMACSHA256(
  base64(header) + "." + base64(payload),
  secretKey
)

If someone modifies the payload (e.g. changes their role to ADMIN), the signature won't match and the server rejects the token.

4. Authentication Flow (Login)

What happens when a user logs in for the first time:

Client
POST /auth/login
{user, pass}

AuthController
->
AuthenticationManager
->
UserDetailsService
(load from DB)
->
BCrypt
password check
->
JwtService
generateToken()
->
Client gets
{ token }

Step-by-step breakdown:

  1. Client sends credentials — POST /api/auth/login with { "username": "alice", "password": "secret" }
  2. AuthController receives request — delegates to AuthenticationService
  3. AuthenticationManager.authenticate() — creates a UsernamePasswordAuthenticationToken and hands it to the provider
  4. DaoAuthenticationProvider — calls UserDetailsService.loadUserByUsername() to fetch user from DB, then uses BCryptPasswordEncoder to compare passwords
  5. If credentials match — JwtService builds the token with claims (sub, roles, iat, exp) and signs it
  6. Response sent to client — { "accessToken": "eyJ..." }
  7. Client stores the token — in memory or an httpOnly cookie

5. Authorization Flow (Protected Requests)

What happens on every subsequent request after login:

Client
GET /api/data
Authorization: Bearer <token>
->
JwtAuthenticationFilter
->
Extract token
from header
->
JwtService
validate token
->
Set
SecurityContext
->
Controller
handles request
  1. Client sends request with header — Authorization: Bearer eyJhbGci...
  2. JwtAuthenticationFilter intercepts — this runs before every request. It reads the Authorization header and strips the "Bearer " prefix
  3. Extract username from token — jwtService.extractUsername(token) parses the JWT and reads the sub claim
  4. Check SecurityContext is empty — if the user is already authenticated this request, skip. Otherwise continue.
  5. Load user from DB — userDetailsService.loadUserByUsername() fetches fresh user data
  6. Validate the token — checks: username matches, token is not expired, signature is valid
  7. Set authentication in SecurityContext — creates UsernamePasswordAuthenticationToken with the user's roles and puts it in SecurityContextHolder
  8. Request proceeds to Controller — Spring Security sees the authenticated user, evaluates @PreAuthorize annotations, and allows the request through
Why does every request re-validate the token?
Because JWT is stateless — the server holds no session. The SecurityContext is cleared after every request. The JWT IS the session.

6. All Components Explained

Class / Component What it does Key methods
JwtService Generates tokens, validates tokens, extracts claims from tokens generateToken(), isTokenValid(), extractUsername()
JwtAuthenticationFilter Runs on every request. Extracts JWT from the Authorization header, validates it, and sets the authentication in SecurityContext doFilterInternal()
SecurityConfig Configures which routes are public vs protected. Adds the JWT filter. Disables sessions. Wires up beans. securityFilterChain()
UserDetailsService Spring Security interface. Your implementation loads the user from the database by username/email loadUserByUsername()
AuthController REST controller that exposes /auth/login and /auth/register endpoints login(), register()
AuthenticationService Business logic for login and registration. Calls AuthenticationManager and JwtService authenticate(), register()
AuthenticationManager Spring Security bean that delegates credential verification to the AuthenticationProvider authenticate()
DaoAuthenticationProvider Built-in Spring class. Uses UserDetailsService to fetch user and PasswordEncoder to compare passwords (auto-used by AuthManager)
PasswordEncoder BCrypt hashing. Hashes passwords on register, compares on login encode(), matches()
SecurityContextHolder Thread-local storage for the current authenticated user. Cleared after every request. getContext(), setAuthentication()
User (Entity) JPA entity that also implements UserDetails. Stored in DB, used by Spring Security. getAuthorities(), getUsername()

User.java — Entity that implements UserDetails

@Entity
@Table(name = "users")
@Data @Builder @NoArgsConstructor @AllArgsConstructor
public class User implements UserDetails {

    @Id @GeneratedValue
    private Integer id;
    private String firstName;
    private String lastName;

    @Column(unique = true)
    private String email;       // used as the username
    private String password;    // BCrypt hashed

    @Enumerated(EnumType.STRING)
    private Role role;          // USER or ADMIN

    // Spring Security requires these methods:

    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return List.of(new SimpleGrantedAuthority(role.name()));
    }

    @Override
    public String getUsername() { return email; }

    @Override public boolean isAccountNonExpired() { return true; }
    @Override public boolean isAccountNonLocked()  { return true; }
    @Override public boolean isEnabled()           { return true; }
}

public enum Role { USER, ADMIN }

JwtService.java — Token generation and validation

@Service
public class JwtService {

    @Value("${application.security.jwt.secret-key}")
    private String secretKey;

    @Value("${application.security.jwt.expiration}")
    private long jwtExpiration;

    // Extract the username (subject) from a token
    public String extractUsername(String token) {
        return extractClaim(token, Claims::getSubject);
    }

    // Generate a token for a user
    public String generateToken(UserDetails userDetails) {
        return Jwts.builder()
                .subject(userDetails.getUsername())
                .issuedAt(new Date(System.currentTimeMillis()))
                .expiration(new Date(System.currentTimeMillis() + jwtExpiration))
                .signWith(getSignInKey(), Jwts.SIG.HS256)
                .compact();
    }

    // Check if the token is valid (correct user + not expired)
    public boolean isTokenValid(String token, UserDetails userDetails) {
        final String username = extractUsername(token);
        return username.equals(userDetails.getUsername()) && !isTokenExpired(token);
    }

    private boolean isTokenExpired(String token) {
        return extractClaim(token, Claims::getExpiration).before(new Date());
    }

    private <T> T extractClaim(String token, Function<Claims, T> resolver) {
        Claims claims = Jwts.parser()
                .verifyWith(getSignInKey())
                .build()
                .parseSignedClaims(token)
                .getPayload();
        return resolver.apply(claims);
    }

    private SecretKey getSignInKey() {
        byte[] keyBytes = Decoders.BASE64.decode(secretKey);
        return Keys.hmacShaKeyFor(keyBytes);
    }
}

JwtAuthenticationFilter.java — Runs on every request

@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtService jwtService;
    private final UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain filterChain) throws ServletException, IOException {

        // 1. Read the Authorization header
        final String authHeader = request.getHeader("Authorization");

        // 2. If no token, skip (public endpoints pass through, protected ones get 403)
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        // 3. Extract the token (remove "Bearer " prefix)
        final String jwt = authHeader.substring(7);
        final String username = jwtService.extractUsername(jwt);

        // 4. If username found and user not yet authenticated this request
        if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {

            UserDetails userDetails = userDetailsService.loadUserByUsername(username);

            // 5. Validate the token
            if (jwtService.isTokenValid(jwt, userDetails)) {

                // 6. Create auth token and set in SecurityContext
                UsernamePasswordAuthenticationToken authToken =
                    new UsernamePasswordAuthenticationToken(
                        userDetails,
                        null,                           // no credentials needed post-auth
                        userDetails.getAuthorities()    // roles
                    );
                authToken.setDetails(
                    new WebAuthenticationDetailsSource().buildDetails(request)
                );
                SecurityContextHolder.getContext().setAuthentication(authToken);
            }
        }

        // 7. Continue the filter chain
        filterChain.doFilter(request, response);
    }
}

SecurityConfig.java — Wire everything together

@Configuration
@EnableWebSecurity
@EnableMethodSecurity   // enables @PreAuthorize on controllers
@RequiredArgsConstructor
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthFilter;
    private final UserDetailsService userDetailsService;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)  // not needed for stateless JWT
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()   // login/register = public
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()                  // everything else needs token
            )
            .sessionManagement(sess ->
                sess.sessionCreationPolicy(SessionCreationPolicy.STATELESS) // no sessions!
            )
            .authenticationProvider(authenticationProvider())
            // JWT filter runs BEFORE Spring's default username/password filter
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public AuthenticationProvider authenticationProvider() {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
        provider.setUserDetailsService(userDetailsService);
        provider.setPasswordEncoder(passwordEncoder());
        return provider;
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public AuthenticationManager authenticationManager(AuthenticationConfiguration config)
            throws Exception {
        return config.getAuthenticationManager();
    }
}

AuthController.java — Login and Register endpoints

@RestController
@RequestMapping("/api/auth")
@RequiredArgsConstructor
public class AuthController {

    private final AuthenticationService authService;

    @PostMapping("/register")
    public ResponseEntity<AuthResponse> register(@RequestBody RegisterRequest request) {
        return ResponseEntity.ok(authService.register(request));
    }

    @PostMapping("/login")
    public ResponseEntity<AuthResponse> login(@RequestBody AuthRequest request) {
        return ResponseEntity.ok(authService.authenticate(request));
    }
}

// DTOs
record AuthRequest(String email, String password) {}
record RegisterRequest(String firstName, String lastName, String email, String password) {}
record AuthResponse(String accessToken) {}

AuthenticationService.java — Business logic

@Service
@RequiredArgsConstructor
public class AuthenticationService {

    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;
    private final JwtService jwtService;
    private final AuthenticationManager authenticationManager;

    public AuthResponse register(RegisterRequest request) {
        var user = User.builder()
                .firstName(request.firstName())
                .lastName(request.lastName())
                .email(request.email())
                .password(passwordEncoder.encode(request.password()))
                .role(Role.USER)
                .build();
        userRepository.save(user);
        var token = jwtService.generateToken(user);
        return new AuthResponse(token);
    }

    public AuthResponse authenticate(AuthRequest request) {
        // Throws BadCredentialsException if wrong password
        authenticationManager.authenticate(
            new UsernamePasswordAuthenticationToken(request.email(), request.password())
        );
        var user = userRepository.findByEmail(request.email()).orElseThrow();
        var token = jwtService.generateToken(user);
        return new AuthResponse(token);
    }
}

    @GetMapping("/all")

        // admin only
    }

8. Security Best Practices

Topic Recommendation
Secret key length Use at least 32 characters (256 bits). Generate with: openssl rand -hex 32
Token expiry Access token: 15 min. Refresh token: 7 days. Never use "no expiry".
Token storage Use httpOnly cookies (safe from XSS). Avoid localStorage — any JS on the page can read it.
Payload contents Only store non-sensitive data (username, roles). Never store passwords or PII.
Secret key location Never hardcode in source code. Use environment variables or a secrets manager.
Algorithm HS256 is fine for single apps. Use RS256 for microservices (share public key only).
Logout / revocation JWT cannot be invalidated early by default. Use a Redis blacklist to support proper logout.
HTTPS Always use HTTPS in production. A JWT sent over plain HTTP can be stolen in transit.
Common mistake: Setting token expiry to 30 days or more "for convenience". If a token is stolen, the attacker has 30 days of access. Use short-lived access tokens + refresh tokens instead.
Refresh Token Pattern: Access token expires in 15 min. Client sends refresh token to POST /auth/refresh. Server issues a new access token. This way tokens are short-lived but users stay logged in.
